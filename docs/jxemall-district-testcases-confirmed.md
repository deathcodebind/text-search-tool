# jxemall 地区筛选测试用例（已确认部分）

## 命名格式规范
- 名称备注统一使用“上级单位 + 下级单位”的完整写法，例如“南昌市西湖区”“吉安市吉安县”。
- 本级统一写作“{市名/省名}本级”，虚拟区级单位保留平台官方名称并带所属上级单位前缀。

## 1. 单选场景
- 只选一个区级单位：
  - ["360102"] // 南昌市东湖区
- 只选市本级虚拟码：
  - ["360699"] // 鹰潭市本级
- 只选虚拟区级单位：
  - ["360992"] // 宜春市明月山温泉风景名胜区
  - ["361209"] // 赣江新区中医药科创城
- 只选省级虚拟单位下属的虚拟市级单位：
  - ["980701"] // 江西培训市
- 只选省本级：
  - ["369900"] // 江西省本级
- 余江区编码特例：
  - ["360622"] // 鹰潭市余江区（特例）

## 4. 约束与负例场景
- 不纳入用例：同市内“市级码 + 下级码”混选
  - 原因：选择市级码后会直接全选该市全部区县与市本级，语义已被上卷，无法形成独立边界。
- 负例：跨市多选含重复码（应判定为不正确）
  - ["360103","360103","360321","360322"] // 南昌市西湖区重复
  - 预期：校验失败或在入参校验阶段拒绝。
- 顺序约束建议：编码数组保持稳定顺序
  - 说明：暂无 API 文档确认顺序是否强制，但建议所有正例统一按固定顺序输出，避免潜在兼容问题。
