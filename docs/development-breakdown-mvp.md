# 全文搜索竞标助手 开发任务拆解（MVP）

## 1. 迭代目标

- 完成从登录、抓取、去重、索引、检索到结果展示的闭环。
- 支持 Windows + macOS 双平台首发。
- 达到 MVP 验收口径：完整用户流程可跑通，且无明显错误。

## 2. 里程碑与交付物

### 2.1 M0 技术验证（建议 1-2 周）
- 完成 jxemall.com 登录态获取 PoC。
- 完成增量抓取最小链路（按时间窗口）。
- 完成 Tantivy + 中文分词最小检索样例。
- 输出：PoC 报告（可行性、风险点、技术决策）。

### 2.2 M1 MVP 可用版（建议 3-5 周）
- 完成多 crates 主体开发。
- 完成 GUI 关键页面与主流程。
- 完成双平台打包可安装。
- 输出：可试用安装包 + 使用说明。

### 2.3 M2 试用迭代（建议 1-2 周）
- 修复试用问题，优化规则体验与稳定性。
- 补齐关键回归测试。
- 输出：MVP 稳定版。

## 3. 按 Crate 拆解

### 3.1 crates/crawler
- 任务：
  - 登录流程与会话维持。
  - 基于站点筛选项的抓取请求构建。
  - 增量时间窗口抓取调度（上次时间点 -> 当前时间点）。
  - 字段映射与标准化输出。
- 交付标准：
  - 可稳定抓取并输出结构化数据。
  - 登录失效可检测并给出可读错误。

### 3.2 crates/storage
- 任务：
  - 元数据模型与持久化层。
  - 去重逻辑（来源 ID 主键，URL 辅键）。
  - 3 天周期过期清理任务。
  - 抓取任务状态记录（开始、进行中、成功、失败）。
- 交付标准：
  - 重复数据不重复入库。
  - 过期清理可执行且可追踪。

### 3.3 crates/search-engine
- 任务：
  - Tantivy 索引结构与字段权重定义。
  - Bool Query 核心子集解析与执行。
  - 评分模型实现与参数配置。
  - 中文分词接入（jieba-rs / tantivy-jieba 选型落地）。
- 交付标准：
  - 规则查询结果可解释（命中词、评分来源）。
  - 查询目标延迟 300ms 以内（后续压测验证）。

### 3.4 crates/gui（Tauri + Vue3 + Pinia）
- 任务：
  - 页面：数据源配置、抓取任务、规则管理、结果列表与详情。
  - Pinia 状态分层：session、crawler、rules、search-results。
  - 用户交互状态提示：加载、失败重试、空结果提示。
  - 系统凭证与会话状态展示。
- 交付标准：
  - 无阻断主流程的关键 UI 错误。
  - 用户可独立完成完整链路操作。

### 3.5 crates/shared（可选）
- 任务：
  - 公共 DTO、错误码、配置结构。
  - 统一时间与分页模型。
- 交付标准：
  - 跨 crate 接口一致且可复用。

## 4. 前后端接口契约（建议先行）

- crawler -> storage：
  - upsert_records(records)
  - mark_job_status(job_id, status, message)
- search -> gui：
  - search(query_rule, paging, sort)
  - get_result_detail(record_id)
- gui -> crawler：
  - start_crawl(source, filters, credential_ref)
  - get_crawl_progress(job_id)

建议先出一版接口草案文档，避免并行开发时反复改动。

## 5. 测试清单（优先补新增功能）

### 5.1 单元测试
- 规则解析：must/should/must_not/minimum_should_match。
- 评分函数：参数变化对结果排序的影响。
- 去重逻辑：主辅键冲突处理。
- 过期清理：边界时间点数据清理正确性。

### 5.2 集成测试
- 抓取 -> 入库 -> 索引 -> 查询完整链路。
- 登录失败、抓取失败、索引失败场景回滚与提示。

### 5.3 GUI 冒烟测试
- 首次配置与登录流程。
- 抓取任务发起、进度查看、结果展示。
- 规则修改后重新查询。

## 6. 风险与预案

- 站点结构变化：
  - 预案：适配器隔离 + 可快速热修字段映射。
- 登录风控策略变化：
  - 预案：会话过期检测 + 手动重新登录流程。
- 分词精度不足：
  - 预案：词典可配置与业务词快速追加。

## 7. 建议的首批实现顺序

1. crawler 最小可用链路 + storage 入库与去重。
2. search-engine 最小查询链路（含中文分词）。
3. GUI 串通主流程。
4. 过期清理与任务可观测性。
5. 双平台打包与安装验证。
