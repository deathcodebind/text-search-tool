# 全文搜索竞标助手 PRD（MVP）

## 1. 产品概述

### 1.1 产品定位
面向公开竞标信息的本地桌面工具，帮助用户通过“两轮筛选”快速定位可承接标的。

### 1.2 核心价值
- 降低信息分散导致的检索成本。
- 通过规则化关键词查询提升筛选效率与可解释性。
- 在本地完成数据处理，保证使用门槛低、流程可控。

## 2. 用户与场景

### 2.1 目标用户
- 对竞标有兴趣的个人或企业用户。
- 需要按行业能力快速发现可参与项目的用户。

### 2.2 典型流程
1. 选择目标网站（MVP 仅 jxemall.com）。
2. 输入登录信息并建立会话。
3. 使用站点原生筛选项做第一轮过滤。
4. 执行增量抓取并等待入库。
5. 配置关键词规则并执行第二轮全文查询。
6. 浏览结果并决定是否参与竞标。

## 3. 功能需求

### 3.1 必须实现
- 登录态管理与站点数据抓取。
- 每日固定时刻增量抓取任务。
- 去重（主键：来源 ID；辅键：URL）。
- 数据过期清理（3 天周期）。
- Tantivy 全文索引与查询。
- 中文分词（jieba-rs / tantivy-jieba 评估后定）。
- Bool Query 核心子集：must/should/must_not/minimum_should_match。
- 强弱关键词权重与可配置评分参数。
- GUI 全流程：配置、拉取、查询、结果查看。
- Tauri 打包，覆盖 Windows 与 macOS 双平台首发。

### 3.2 结果展示（初版）
- 标题
- 发布时间
- 地区
- 来源链接
- 命中词/命中摘要

### 3.3 非目标（MVP 不做）
- 多用户与权限体系。
- 云同步与团队协作。
- 多站点并行适配（除 jxemall.com 之外）。
- 完整 Bool Query 全量语法（视反馈再扩展）。

## 4. 非功能需求

### 4.1 性能
- 查询延迟目标：300ms 以内（目标值）。
- 性能压测在主要功能反馈迭代后执行。

### 4.2 可用性
- MVP 验收标准：完整用户流程可跑通，且无明显错误。

### 4.3 合规与安全
- 仅在用户具备合法登录身份前提下抓取公开信息。
- 凭证存储优先系统 Keychain（或浏览器会话模型）。

### 4.4 存储约束
- 元数据存储采用 SQLite（MVP 确定），配合 Tantivy 承担全文检索。
- 过期清理采用基于 expires_at 索引的定时批处理删除。

## 5. 验收标准

### 5.1 功能验收
- 可从登录到抓取、从索引到检索完成闭环。
- 可配置关键词规则并获得符合预期的筛选结果。
- 可查看结果详情并跳转来源链接。

### 5.2 稳定性验收
- 连续多次抓取任务可稳定执行。
- 去重与过期清理生效，数据状态可预期。

## 6. 里程碑

- M0：检索与抓取链路技术验证。
- M1：MVP 可试用版（端到端闭环）。
- M2：基于试用反馈的稳定性与体验迭代。
