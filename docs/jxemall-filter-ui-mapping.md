# jxemall 筛选项 UI 映射表（MVP）

## 1. 用途

本表用于统一三件事：
- 页面筛选项与请求 payload 字段的映射
- 已确认值与待验证值的区分
- GUI 给用户提供筛选表单时的默认行为

证据来源：你提供的页面截图 + 已确认 payload 样本。

## 2. 筛选项映射（按页面分组）

### 2.1 采购内容

| 页面文案 | payload 字段 | 值 | 状态 |
|---|---|---|---|
| 全部 | categoryType | null（需显式传） | 已确认 |
| 货物类 | categoryType | GOODS | 已确认 |
| 服务类 | categoryType | SERVICE | 已确认 |
| 工程类 | categoryType | PROJECT | 已确认 |

说明：
- 已确认“全部时为 null”，且需显式传该字段。
- 已确认分类编码：GOODS/SERVICE/PROJECT。
- 建议在项目内部定义无歧义中间枚举，再经 serde 序列化为上述字符串。
- 语义以中文筛选项为准：`PROJECT` 对应“工程类”，仅为后端命名词，不改变我方业务语义命名。

### 2.2 当前状态

| 页面文案 | payload 字段 | 值 | 状态 |
|---|---|---|---|
| 全部 | stateList | [3, 4, 5, 6, 7, 10, 12, 50] | 已确认 |
| 竞价未开始 | stateList | [3] | 已确认 |
| 竞价中 | stateList | [4] | 已确认 |
| 已过期 | stateList | [5, 6, 7, 10, 12, 50] | 已确认 |

说明：
- 当前状态筛选是对后端状态码集合的映射。
- 已确认：3=未开始，4=竞价中，5/6/7/10/12/50 为已结束集合。
- 产品约束：GUI 不直接展示状态码，只展示业务文案（未开始/竞价中/已过期）。
- 实现建议：定义中间枚举 `All/NotStarted/InProgress/Expired`，由映射函数统一生成 `stateList`。

### 2.4 控制总价

| 页面文案 | payload 字段 | 值 | 状态 |
|---|---|---|---|
| 全部 | minBudget/maxBudget | 不传（不要传 null） | 已确认 |
| 5万以下 | maxBudget | 50000 | 已确认 |
| 5-6万 | minBudget + maxBudget | 50000 + 60000 | 已确认 |
| 6-7万 | minBudget + maxBudget | 60000 + 70000 | 已确认 |
| 10万以上 | minBudget | 100000 | 已确认 |

说明：
- 已确认控制总价使用额外字段 `minBudget` / `maxBudget`（数字）。
- “区间”语义使用双边界；“仅上限/仅下限”语义分别只传 maxBudget 或 minBudget。
- 实现规范：全部场景省略可选字段，不传 null。
- 实现规范：
- `categoryType` 全部时显式传 `null`。
- `minBudget`/`maxBudget` 全部时省略字段，不传 `null`。

## 4. GUI 表单默认建议（用于 MVP）

- 采购内容默认：全部（`categoryType = null`）。
- 当前状态默认：全部（stateList = [3, 4, 5, 6, 7, 10, 12, 50]）。
- 业务实例默认：使用页面顺序全选 `["JXWSCS","JXDDCG","JXXYGH","JXFWGC"]`。
- 地区默认：不限制（两个列表均为空数组）。
- 若用户执行“全选地区”，应传递全选代码集，不应回写为空数组。
- 排序默认：公告发布时间 DESC。
- 控制总价默认：全部（不传 minBudget/maxBudget）。

交互约束：
- 用户侧筛选控件只展示业务状态文案，内部再映射为 `stateList` 数组。
- 避免在界面和帮助文案中暴露 `3/4/5/6/7/10/12/50` 等实现细节。

## 5. 数据结构建议（前端）

```ts
type VerificationStatus = "confirmed" | "pending";

interface FilterOption {
  label: string;
  payloadField: string;
  payloadValue: unknown;
  status: VerificationStatus;
}
```
